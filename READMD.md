## 打卡模組

### 管理員設定（打卡）

進入後台 → 左側選單「**打卡管理**」

#### ① 新增員工

**打卡管理 → 員工管理 → 新增員工**

| 欄位 | 說明 |
|------|------|
| 姓名 | 顯示用名稱 |
| 登入帳號 | 員工用於登入打卡頁的帳號（英文或數字）|
| 密碼 | 至少 4 個字元 |
| 職稱 | 選填，例：外場、廚房 |
| 狀態 | 在職 / 停用 |

> 停用的員工無法登入打卡頁。

---

#### ② GPS 地點設定

**打卡管理 → GPS 設定**

1. 點「**新增地點**」填入：
   - 地點名稱（例：公司門口）
   - 緯度 / 經度（點「取得目前位置」自動填入，或從 Google Maps 右鍵複製）
   - 允許範圍（公尺）：員工打卡時距離此地點的最大距離
2. 上方選擇「GPS 驗證模式」：
   - **不強制**：記錄員工距離，但不阻擋打卡
   - **強制**：超出所有地點的允許範圍則打卡失敗

> 可設定多個地點，員工在任一地點範圍內即可打卡。

---

#### ③ 出勤摘要查詢

**打卡管理 → 出勤摘要**

- 選擇月份，查看每位員工每天的：
  - 上班時間、下班時間
  - 工時（小時分鐘）
  - 是否含補打卡記錄

---

#### ④ 打卡記錄管理

**打卡管理 → 打卡記錄**

可依員工、日期區間、月份篩選，支援：
- **編輯**：修改打卡時間、類型、備註
- **刪除**：刪除單筆打卡記錄

---

#### ⑤ 管理員補打卡

**打卡管理 → 補打卡（管理員）**

由管理員直接幫員工新增補打卡記錄，填入：
- 員工姓名
- 打卡類型（上班 / 下班 / 休息開始 / 休息結束）
- 補打時間
- 操作人員、備註

---

#### ⑥ 審核員工補打卡申請

**打卡管理 → 補打申請**

員工自行送出的補打卡申請，管理員可：

| 操作 | 說明 |
|------|------|
| 核准 | 系統自動建立補打卡記錄，員工打卡記錄同步更新 |
| 退回 | 填寫退回原因，員工在員工頁可看到說明 |

預設顯示「待審核」，可切換查看全部 / 已核准 / 已退回。

---

#### ⑦ LINE 官方帳號打卡設定

**打卡管理 → LINE 打卡**

**首次設定步驟：**

1. 至 [LINE Developers](https://developers.line.biz) 建立 Messaging API Channel
2. 複製 **Channel Secret** 和**長效 Channel Access Token**
3. 填入後台，點「儲存設定」並選擇「啟用」
4. 複製頁面顯示的 **Webhook URL**，填入 LINE Developers → Webhook URL，並開啟「Use Webhook」
5. 關閉「Auto-reply messages」和「Greeting messages」（避免系統訊息衝突）

**建立圖文選單（快速打卡按鈕）：**

1. 點「**建立圖文選單**」
2. 員工重新開啟 LINE 聊天室，底部即出現四個彩色打卡按鈕

**員工 LINE 綁定狀態：**

顯示每位員工是否已完成 LINE 帳號綁定，可強制解除綁定。

---

### 員工操作（網頁打卡）

網址：`https://your-app.onrender.com/staff`

**登入：**
- 輸入管理員設定的帳號及密碼

**打卡 Tab：**

1. 頁面顯示 GPS 定位狀態（綠色=範圍內，橘色=超出範圍）
2. 點擊「**上班打卡**」「**下班打卡**」「**休息開始**」「**休息結束**」
3. 打卡後畫面顯示綠色確認動畫
4. 下方「今日打卡記錄」即時更新

**申請補打卡：**

1. 打卡記錄下方點「**申請補打卡**」
2. 填入：打卡類型、補打時間、原因（必填）
3. 送出後等待管理員審核
4. 申請狀態顯示：審核中 / 已核准 / 已退回（含退回原因）

---

### 員工操作（LINE 打卡）

**首次綁定（只需做一次）：**

1. 掃描 LINE 官方帳號 QR Code 加入好友
2. 在聊天室輸入：`綁定 你的帳號`
   - 例：`綁定 mary123`
   - 帳號為管理員設定的**登入帳號**，不知道請詢問管理員

**打卡方式：**

| 方式 | 說明 |
|------|------|
| 圖文選單按鈕 | 點底部彩色按鈕（上班打卡 / 下班打卡 / 休息開始 / 休息結束）|
| 傳送位置訊息 | 點 ＋ → 位置資訊 → 自動偵測打卡類型並驗證 GPS |
| 文字指令 | 輸入「上班」「下班」「休息」「回來」|

**查詢今日記錄：**
- 輸入「**狀態**」→ 顯示今日所有打卡時間

**解除綁定：**
- 輸入「**解除綁定**」

> 若開啟強制 GPS 驗證，點按鈕後系統會要求傳送位置，點畫面下方「傳送我的位置」即可完成。

## 排班模組

### 管理員設定（排班）

進入後台 → 左側選單「**排班管理**」

#### ① 月份設定

**排班管理 → 月份設定**

每月開放排休前先設定：

| 欄位 | 說明 | 預設 |
|------|------|------|
| 每日最多休假人數 | 同一天最多幾人可排休 | 2 人 |
| 每人每月休假配額 | 員工每月可申請幾天排休 | 8 天 |
| 備註 | 員工開啟排休頁時會彈出提示 | — |

> 例：設定「每日最多 2 人」，當天已有 2 人申請，第 3 人選擇同一天會直接被擋下。

---

#### ② 月曆總覽

**排班管理 → 月曆總覽**

選擇月份，以月曆形式查看：
- 每天誰在休假（綠色 = 已核准，橘色 = 待審中）
- 是否有超出人數上限（紅色警示）

右上角「**員工系統**」連結可快速開啟員工打卡/排班頁。

---

#### ③ 審核排休申請

**排班管理 → 排休申請**

可依月份和狀態篩選，針對每筆申請：

| 操作 | 說明 |
|------|------|
| 審核（核准）| 申請狀態變為「已核准」，員工端顯示綠色 |
| 審核（退回）| 填寫退回原因，員工需重新選取日期送出 |
| 刪除 | 永久刪除此申請記錄 |

> 員工修改已核准的申請後，狀態變為「修改待審」，需重新審核。

---

#### ④ 人員摘要

**排班管理 → 人員摘要**

選擇月份，一覽所有員工：
- 申請幾天 / 配額幾天
- 目前狀態（未申請 / 審核中 / 已核准 / 已退回）
- 申請的具體日期

---

### 員工操作（排班）

員工系統網址：`https://your-app.onrender.com/staff`（用打卡帳號登入）

**排休申請 Tab：**

1. 選擇要申請的月份（預設顯示下個月）
2. 上方顯示「已選天數 / 本月配額」
3. 點擊月曆上的日期選取或取消排休日
   - 灰色日期 = 過去日期（不可選）
   - 紅字「額滿」= 該天人數已達上限（不可選）
   - 其他日期上顯示已有幾人申請
4. 可填入備註（例：家庭活動）
5. 點「**送出申請**」→ 確認日期清單 → 確認送出

**申請注意事項：**
- 超過本月配額時自動阻擋，無法多選
- 選到已額滿的日期會即時提示
- 如果原本有已核准的申請，修改後需再次等待審核

**我的排班 Tab：**

選擇月份，以月曆顯示當月休假狀況：
- 綠色 = 核准休假
- 橘色 = 待審中
- 紅色 = 已退回（需重新申請）

## 薪資模組

### 管理員設定（薪資）

進入後台 → 左側選單「**薪資管理**」

#### ① 薪資項目設定

**薪資管理 → 薪資項目**

設定每月薪資計算使用的津貼和扣除項目。

**系統預設項目（可修改）：**

| 項目 | 類型 | 計算方式 |
|------|------|----------|
| 本薪 | 津貼 | `base_salary`（帶入底薪）|
| 伙食津貼 | 津貼 | 固定 $2,400 |
| 職務加給 | 津貼 | 固定（預設 0，可自行輸入）|
| 全勤獎金 | 津貼 | 固定（預設 0）|
| 加班費 | 津貼 | 固定（預設 0）|
| 獎金 | 津貼 | 固定（預設 0）|
| 勞保費 | 扣除 | `insured_salary × 1.1%` |
| 健保費 | 扣除 | `insured_salary × 5.17% ÷ 6 × 2` |
| 勞退 6% | 津貼（參考）| `base_salary × 6%` |

**新增自訂項目：**
- 名稱、類型（津貼 / 扣除）
- 計算方式選「固定金額」或「公式計算」
- 公式可用變數：
  - `base_salary`（底薪）
  - `insured_salary`（投保薪資）
  - `service_years`（年資，整數）
- 勾選「**生日禮金**」= 只有當月生日的員工才會計入此項目

---

#### ② 員工薪資設定

**薪資管理 → 薪資設定**

員工清單來自「打卡管理 → 員工管理」，無需重複新增。
點「**設定薪資**」為每位員工填入：

| 欄位 | 說明 |
|------|------|
| 員工代碼 | 薪資單上的編號，例：EMP001 |
| 部門 | 例：外場、廚房 |
| 職稱 | 例：服務員、領班 |
| 到職日 | 用於計算年資 |
| 生日 | 用於判斷是否發放生日禮金 |
| 底薪（月）| 月薪基本薪資 |
| 投保薪資 | 用於勞保 / 健保公式計算（依勞保級距填入）|

---

#### ③ 月薪計算

**薪資管理 → 月薪計算**

**自動產生薪資：**
1. 選擇月份
2. 點「**自動產生薪資**」
3. 系統依薪資項目設定，為所有在職員工自動計算每一項津貼與扣除
4. 表格顯示每位員工：津貼合計、扣除合計、實領金額

> 已確認的薪資記錄不會被自動產生覆蓋。

**編輯薪資明細：**
1. 點某員工的「**編輯**」
2. 可新增 / 修改 / 刪除各津貼或扣除項目
3. 修改金額後總額即時重算
4. 填入支付日期和備註
5. 點「儲存」

**確認薪資：**
- 點「**確認**」鎖定薪資記錄
- 確認後員工可在員工系統查詢
- 確認後不會被自動覆蓋

**列印薪資單：**
- 在編輯視窗點「**列印薪資單**」
- 開啟新視窗，顯示格式化的薪資單（含明細），可直接列印或另存 PDF

薪資單格式：

```
2026/03 薪資明細表
員工代碼  EMP001       姓名   王小明
部門      外場         支付日期  2026/04/05

支付總額 $33,400  扣除總額 $1,810  實支付額 $31,590

津貼項目
  本薪          $30,000
  伙食津貼       $2,400
  全勤獎金       $1,000
  合計          $33,400

扣除項目
  勞保費         $1,100
  健保費           $710
  合計           $1,810
```

## 環境變數設定

在 Render Dashboard → Web Service → Environment 設定以下變數：

| 變數名稱 | 說明 | 必填 |
|----------|------|------|
| `DATABASE_URL` | PostgreSQL 資料庫連線（Internal URL）| ✅ |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Bot（記帳用）Token | ✅ |
| `LINE_CHANNEL_SECRET` | LINE Bot（記帳用）Secret | ✅ |
| `ADMIN_PASSWORD` | 後台登入密碼 | ✅ |
| `OPENAI_API_KEY` | OpenAI API Key（AI 報帳辨識用）| 選填 |
| `SECRET_KEY` | Flask Session 加密金鑰（Render 自動產生）| ✅ |

> **LINE 打卡** 的 Channel Token 和 Secret 在後台「打卡管理 → LINE 打卡」頁面設定，不需要環境變數。

## 部署說明

### 首次部署

```bash
# 1. Fork 或 Clone 此 Repository
git clone https://github.com/your-repo/linebot-finance.git

# 2. Push 到 GitHub
git add .
git commit -m "initial commit"
git push origin main
```

在 Render Dashboard：
1. **New → PostgreSQL** 建立資料庫（Free tier）
2. **New → Web Service** 連結 GitHub Repo
3. 設定環境變數（見上方表格）
4. **DATABASE_URL** 從 PostgreSQL → Info → **Internal Database URL** 複製
5. Deploy 完成後設定 LINE Webhook URL：
   `https://your-app.onrender.com/webhook`

### 更新部署

```bash
git add .
git commit -m "update"
git push origin main
```

Render 會自動重新部署（約 2-3 分鐘）。

## 常見問題

**Q：員工忘記打卡怎麼辦？**
員工可在員工系統「打卡 Tab → 申請補打卡」送出申請，管理員在後台「補打申請」審核後系統自動補打。

**Q：LINE 綁定帳號顯示「找不到帳號」？**
請確認輸入的是打卡帳號（英文字母），不是說明文字「帳號」。正確格式：`綁定 mary123`

**Q：打卡顯示超出範圍？**
確認 GPS 是否已允許定位。若地點設定正確但距離顯示異常，可調大「允許範圍」公尺數，或確認 GPS 座標是否正確。

**Q：員工看不到薪資？**
薪資需要管理員在後台「月薪計算」頁面點「確認」後才對員工可見。草稿狀態不顯示。

**Q：排休申請顯示「已退回」？**
重新點選日期（可修改原本的日期），再次送出，管理員重新審核即可。
